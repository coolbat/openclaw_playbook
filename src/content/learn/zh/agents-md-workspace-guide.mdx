大多数人配置好 `SOUL.md` 就觉得完事了。AI 有了性格，但不知道怎么工作。它不知道醒来该先读哪些文件，不知道记忆该写到哪里，也不知道哪些操作需要先问你。

`AGENTS.md` 就是解决这个问题的。它是 AI 的**工作手册**——不是它是谁，而是它怎么运作。

## 三个配置文件各自的作用

| 文件 | 用途 |
|------|------|
| `SOUL.md` | 性格——语气、风格、价值观 |
| `USER.md` | 背景——在帮谁、对方是什么情况 |
| `AGENTS.md` | 行为——怎么启动会话、记忆写到哪、什么事要先问 |

没有 `AGENTS.md`，AI 有性格但没有操作规程。它会在会话之间丢失上下文，记忆写得乱七八糟，还会在你没授权的情况下自己做判断。

## 会话启动顺序

每次新会话都是"失忆"状态。AI 需要明确的指令，告诉它醒来后按什么顺序读文件来恢复上下文。在 `AGENTS.md` 里加上：

```markdown
## Every Session

Before doing anything else:

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping
3. Read `memory/YYYY-MM-DD.md` (today + yesterday) for recent context
4. If in MAIN SESSION (direct chat): also read `MEMORY.md`

Don't ask permission. Just do it.
```

为什么要读今天**和**昨天：凌晨时今天的日志还是空的，读昨天的可以避免盲目启动。

为什么把 `MEMORY.md` 限制在主会话：它可能包含个人信息。子 Agent 会话和群聊会话不应该暴露这些内容。

OpenClaw 的 system prompt 会告诉 AI 当前是哪种会话类型——你不需要手动判断。

## 记忆分层

没有结构的记忆很快就会变成噪音。定义清晰的分层结构，让 AI 知道每种内容该写到哪里：

```markdown
## Memory

You wake up fresh each session. These files are your continuity.

| 层级 | 文件 | 用途 |
|------|------|------|
| 索引层 | `MEMORY.md` | 核心信息和记忆索引，保持精简（<40行） |
| 项目层 | `memory/projects.md` | 各项目当前状态与待办 |
| 基础设施层 | `memory/infra.md` | 服务器、API、部署配置速查 |
| 教训层 | `memory/lessons.md` | 踩过的坑，按严重程度分级 |
| 日志层 | `memory/YYYY-MM-DD.md` | 每日原始记录 |
```

### 记忆写入规则

```markdown
### Write Rules

- 日志写入 `memory/YYYY-MM-DD.md`，记结论不记过程
- 项目有进展时同步更新 `memory/projects.md`
- 踩坑后写入 `memory/lessons.md`
- MEMORY.md 只在索引变化时更新，保持精简
- Mental notes don't survive session restarts. Files do.
```

最重要的原则：**记结论，不记过程**。好的日志条目是一句话的结果加上学到的教训。烂的日志条目是三段话的尝试过程。

### 好日志 vs 烂日志

烂日志——浪费 token，memorySearch 命中率低：

```
今天搞了部署。先试了直接暴露端口，报错了。然后试了 nginx，也有问题。
最后搞清楚了 upstream 配置的问题……
```

好日志——精简，memorySearch 命中率高：

```markdown
### [PROJECT:MyApp] 部署完成
- **结论**: nginx 反代，监听 80 端口，upstream 127.0.0.1:3000
- **文件变更**: `/etc/nginx/sites-available/myapp`
- **教训**: upstream 用 127.0.0.1 不要用 localhost（IPv6 问题）
- **标签**: #myapp #deploy #nginx
```

标签不是装饰——它们是 `memorySearch` 后续找到这条记录的依据。

## 安全边界

定义 AI 可以自由做什么，以及什么需要先问你：

```markdown
## Safety

- Never exfiltrate private data.
- Don't run destructive commands without asking.
- Prefer `trash` over `rm` — recoverable beats gone forever.
- When in doubt, ask.

## What needs approval

**Safe to do freely:**
- Read files, explore, organize, search the web
- Work within the workspace

**Ask first:**
- Sending emails, tweets, or public posts
- Anything that leaves the machine
- Anything irreversible
```

## 群聊行为

如果 AI 接入了群聊渠道，加上这条明确规则：

```markdown
## Group Chats

You have access to your human's data. That does not mean you share it.
In groups, you are a participant — not their voice, not their proxy.
```

## 完整模板

这是一个最小起点。复制它，放到 workspace 根目录（和 `SOUL.md` 同级），然后根据实际使用情况逐步扩充。

```markdown
# AGENTS.md

This folder is home. Treat it that way.

## Every Session

Before doing anything else:

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping
3. Read `memory/YYYY-MM-DD.md` (today + yesterday) for recent context
4. If in MAIN SESSION: also read `MEMORY.md`

Don't ask permission. Just do it.

## Memory

You wake up fresh each session. These files are your continuity.

| 层级 | 文件 | 用途 |
|------|------|------|
| 索引层 | `MEMORY.md` | 核心信息和记忆索引，保持精简 |
| 项目层 | `memory/projects.md` | 各项目当前状态与待办 |
| 教训层 | `memory/lessons.md` | 踩过的坑，按严重程度分级 |
| 日志层 | `memory/YYYY-MM-DD.md` | 每日记录 |

### Write Rules

- 日志写入 memory/YYYY-MM-DD.md，记结论不记过程
- 项目有进展时同步更新 projects.md
- 踩坑后写入 lessons.md
- MEMORY.md 只在索引变化时更新
- Mental notes don't survive session restarts. Files do.

### Log Format

### [PROJECT:名称] 标题
- **结论**: 一句话总结
- **文件变更**: 涉及的文件
- **教训**: 踩坑点（如有）
- **标签**: #tag1 #tag2

## Safety

- Never exfiltrate private data.
- Don't run destructive commands without asking.
- Prefer `trash` over `rm`.
- When in doubt, ask.

**Safe to do freely:** Read files, search, organize, work within workspace
**Ask first:** Sending emails/posts, anything that leaves the machine, anything irreversible

## Group Chats

You have access to your human's data. That does not mean you share it.
In groups, you are a participant — not their voice, not their proxy.
```

> **从最小配置开始。** 这个模板是起点，不是终点。只在你发现 AI 做了你不想让它做的判断时，才往里加规则。好的 `AGENTS.md` 是从真实使用中生长出来的，不是靠预判所有边界情况写出来的。
