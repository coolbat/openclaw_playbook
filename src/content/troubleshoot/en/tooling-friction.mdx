## What this symptom usually means

The core system may be healthy, but the surrounding stack is too complex. Too many tools, each with their own edge cases, create fragility faster than value.

## First recovery loop

Reduce the workflow to one primary destination and one simple handoff format. If that works, the problem is not OpenClaw itself.

## What to verify next

Map out where authentication, formatting, or routing complexity is being introduced. The weakest integration usually sets the reliability ceiling.

## If the issue persists

Prove value in one integration before scaling out. Reliability comes from fewer moving parts, not more adapters.
