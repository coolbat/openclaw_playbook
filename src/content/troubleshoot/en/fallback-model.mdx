## What this symptom usually means

The system may still be operating, but not on the model path you expect. A fallback route can make the workflow feel broken even though requests technically succeed.

## First recovery loop

Inspect the actual model name in logs and compare the outputs with a previously good baseline. Do not trust assumptions about which model is running.

## What to verify next

Check whether the primary model is unavailable, whether provider policy changed, or whether a cheaper path is being selected automatically.

## If the issue persists

Temporarily force one explicit model path for diagnosis. That is often the fastest way to separate routing behavior from prompt behavior.
