## When to use this template

Use this when reading research already happens, but the output does not become reusable team knowledge. The template helps bridge private reading and shared understanding.

## Minimal setup

Choose one paper source and one summary format. A smaller pipeline is better than a broad pipeline that never gets reviewed.

## What good output looks like

The summary should explain the paper's core idea, why it matters to your team, and whether it changes any current assumptions or plans.

## First extension

Add a lightweight comparison layer so new papers can be placed against previous ones instead of being read in isolation.
